# AI.md

Projeto: AURORA - Web Development e Front End Design

Ferramenta utilizada: Codex / OpenAI

Este documento apresenta um resumo consolidado das principais interacoes com IA utilizadas no projeto, agrupadas por funcionalidade.

Este arquivo registra, de forma resumida, o uso de IA como apoio durante o desenvolvimento da entrega.

## 1. Planejamento das funcionalidades JavaScript

O que foi pedido para a IA:

- Orientacao sobre como desenvolver as funcionalidades exigidas no projeto: slideshow, formulario validado, quiz dinamico e temas com JavaScript.

O que a IA retornou:

- Explicacao da estrutura geral do JavaScript.
- Sugestoes de organizacao por funcoes.
- Orientacao sobre como testar cada parte no navegador.

O que foi mantido, alterado ou rejeitado:

- Foi mantida a ideia de separar o codigo por funcionalidades.
- O codigo foi adaptado ao projeto AURORA durante a implementacao.

## 2. Formulario de contato

O que foi pedido para a IA:

- Apoio para criar e validar o formulario da secao "Junte-se a missao AURORA".

O que a IA retornou:

- Sugestao de campos obrigatorios.
- Validacao visual com HTML, CSS e JavaScript.
- Uso de `required`, `type="email"`, `checkValidity()` e classe de erro no formulario.

O que foi mantido, alterado ou rejeitado:

- Foi mantida a validacao visual.
- Foi rejeitado o uso de `alert()` para campos invalidos.
- O visual foi ajustado para seguir o estilo do site.

## 3. Quiz dinamico

O que foi pedido para a IA:

- Apoio para criar o quiz sobre a missao AURORA.

O que a IA retornou:

- Estrutura do quiz em JavaScript.
- Sugestao de perguntas e alternativas.
- Logica de pontuacao, feedback visual e avanco automatico.

O que foi mantido, alterado ou rejeitado:

- Foi mantido o quiz dinamico gerado por JavaScript.
- Foram mantidas as 10 perguntas relacionadas ao projeto.
- A implementacao foi ajustada e revisada durante os testes.

## 4. Ajustes e revisao

O que foi pedido para a IA:

- Apoio para revisar erros de JavaScript, links de navegacao e organizacao dos arquivos.

O que a IA retornou:

- Correcoes pontuais no codigo.
- Sugestoes de mensagens de commit.
- Orientacao sobre revisao antes de enviar ao GitHub.

O que foi mantido, alterado ou rejeitado:

- Foram mantidas apenas as alteracoes consideradas uteis para a entrega.
- O aluno revisou os arquivos antes de commitar e enviar ao repositorio.

## Observacao final

A IA foi utilizada como apoio para orientacao, depuracao e revisao. A implementacao final foi revisada pelo aluno antes da entrega.
